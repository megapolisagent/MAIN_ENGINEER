# Property Analyst — Архитектурный документ

## Назначение

AI-система подготовки профессиональной аналитики объектов недвижимости для клиентов агентства Megapolis. Фокус Stage 1: продавец.

---

## Workflow — 6 слоёв

```
[ВХОД]
Объект (URL Циан / параметры) + Контекст (A/B/C) + Enrichment block

[СЛОЙ 1] Данные объекта        ← Циан
[СЛОЙ 2] Рынок и аналоги       ← Циан
[СЛОЙ 3] Позиция на рынке      ← детерминировано (формулы)
[СЛОЙ 4] Район и локация       ← 2GIS / Яндекс Карты API
[СЛОЙ 5] Экспертная оценка     ← AI: сегмент авто + контекст A/B/C
[СЛОЙ 6] PDF-отчёт             ← один шаблон, адаптивная секция
```

---

## Input Protocol

```
ОБЯЗАТЕЛЬНО:
- URL объявления Циан / параметры объекта
- Контекст продавца:
    A. Нет цены — нужен рекомендованный диапазон
    B. Объект висит без спроса — нужна диагностика
    C. Идут переговоры — нужны аргументы

STRUCTURED ENRICHMENT BLOCK (опционально, но структурировано):
- Ремонт: [описание]
- Вид из окна: [описание]
- Срочность продажи: [да/нет/комментарий]
- Юридические нюансы: [описание]
- Преимущества: [список]
- Проблемы/минусы: [список]

ОПРЕДЕЛЯЕТСЯ СИСТЕМОЙ АВТОМАТИЧЕСКИ:
- Сегмент объекта (по цене/м² и параметрам)
- Аналоги (радиус, тип, комнатность, площадь ±20%)
- Данные локации
```

---

## Output Contract

```
ГАРАНТИРУЕТСЯ:
- PDF: резюме / рынок / позиция / район / адаптивная секция
- Минимальная визуализация: таблица аналогов +
  позиция объекта на шкале цены +
  simple market positioning chart
- Feedback card после каждого отчёта
- Числа в PDF = числа из формульного слоя (слой 3)
- AI-текст строго на данных слоёв 1-4

НЕ ГАРАНТИРУЕТСЯ на Stage 1:
- Скорость (нет SLA)
- Автодоставка клиенту (Мария отправляет руками)

РАСПОЛОЖЕНИЕ ФАЙЛОВ:
AI_Projects/property-analyst/reports/
```

## Deterministic / AI граница

```
СЛОЙ 1 — формула: извлечение полей объекта. AI: нет.
СЛОЙ 2 — формула: фильтры аналогов. AI: нет.
СЛОЙ 3 — формула: цена/м², медиана, отклонение,
          перцентиль, срок экспозиции, сегмент. AI: нет.
СЛОЙ 4 — формула: сбор данных из API.
          AI: интерпретация для данного объекта и сегмента.
СЛОЙ 5 — формула: базовый расчёт рекомендованной цены.
          AI: обоснование, нарратив, аргументы под контекст A/B/C.
СЛОЙ 6 — формула: структура PDF, все числа из слоя 3.
          AI: текстовые блоки.
```

**AI Reasoning Boundaries:**
- Имеет право: интерпретировать, формулировать, ранжировать аргументы, адаптировать под контекст
- Не имеет права: добавлять факты, делать утверждения без источника, придумывать тренды, использовать предположения при отсутствии данных
- При отсутствии данных: некритичный фактор — пропустить молча; критичный — пометить явно

**Критичные данные (отсутствие помечается явно):**
- Аналоги с рынка
- Цена/м² рынка
- Адрес и координаты объекта

---

## Hallucination Control

- Уровень 1: AI получает только структурированные данные слоёв 1-4
- Уровень 2: AI заполняет шаблон с полями, не пишет свободный нарратив
- Уровень 3: cross-check — числа в тексте совпадают с числами слоя 3

---

## Agent Separation

```
Stage 1: один агент — property-analyst
  └─ retrieval layer  (слои 1-2: Циан + 2GIS)
  └─ analyst layer    (слои 3-6: расчёт + AI + PDF)

Stage 2+: retrieval layer отделяется →
  переиспользуется для покупателя и инвестора
```

---

## QA и Review Loop

**Hybrid QA model:**
- Агент генерирует feedback card после каждого отчёта
- Мария отмечает категорию проблемы (минимальный ввод)
- Система маршрутизирует отклонение в нужный слой

**Feedback card:**
```
[ ] Аналоги       →  retrieval layer: фильтры
[ ] Цена          →  formula layer: коэффициенты
[ ] Галлюцинация  →  prompt layer: grounding
[ ] Тон           →  prompt layer: tone-of-voice
[ ] Формулировки  →  prompt layer: narrative
[ ] Другое        →  накапливается, ручной разбор при калибровке
Комментарий: ___
```

Первая калибровка — после 10-20 отчётов.
