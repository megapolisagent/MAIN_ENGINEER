# PROJECT_DNA — property-analyst

## Контекст бизнеса

Агентство премиальной недвижимости Москвы **Megapolis**.
Сегмент: премиум и бизнес-класс (30+ млн ₽).
Владелец: Мария Боголюбова.

Система готовит профессиональные аналитические отчёты по объектам недвижимости для клиентов агентства. Stage 1 фокус: **продавец**.

---

## Пользователь Stage 1

**Продавец** — приходит с одним из трёх контекстов:

| Контекст | Запрос |
|---|---|
| A | Нет цены — нужен рекомендованный диапазон и точка входа |
| B | Объект висит без спроса — нужна диагностика и стратегия снижения |
| C | Идут переговоры — нужны аргументы и пространство торга |

---

## Tone of Voice — Megapolis

- Тихая сила. Интеллектуальный premium.
- Конкретность и точность — без воды.
- Профессиональный нарратив без AI-slop.
- Не демонстративность, не fake luxury.
- Humanizer обязателен перед любым текстом клиенту.

---

## Что система делает

Принимает объект → парсит рынок → считает позицию → анализирует локацию → формирует экспертную оценку → выдаёт PDF.

---

## Что система НЕ делает (Stage 1)

- Не работает с профилями покупателя и инвестора
- Не доставляет PDF автоматически
- Не хранит историю объектов
- Не мониторит рынок в реальном времени
- Не сравнивает несколько объектов одновременно

---

## Репутационные запреты — КРИТИЧНО

Система работает с клиентами агентства. Ошибка в отчёте = репутационный риск.

**Запрещено:**
- Рыночные утверждения без источника данных
- Упоминание трендов без данных из retrieval layer
- Цифры доходности или роста без расчёта
- Ссылки на факты о районе которых нет в 2GIS/Яндекс данных
- Вероятностные предположения при отсутствии данных

**При отсутствии данных:**
- Некритичный фактор — пропустить молча
- Критичный фактор (аналоги, цена/м² рынка, координаты) — пометить явно

---

## Источники данных

| Слой | Источник |
|---|---|
| Объект + аналоги | Циан API / парсинг |
| Локация и инфраструктура | 2GIS API или Яндекс Карты API |
| AI reasoning и нарратив | Claude API |

---

## Output

- **Формат:** PDF
- **Путь:** `AI_Projects/property-analyst/reports/`
- **Структура:** резюме / рынок / позиция / визуализация / район / адаптивная секция (A/B/C)
- **После каждого отчёта:** feedback card для QA
