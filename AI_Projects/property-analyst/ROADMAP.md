# ROADMAP — property-analyst

---

## Stage 1 — Working Prototype

Цель: первый PDF-отчёт по реальному объекту для продавца.

---

### Phase 0 — Manual Proof
**Цель:** убедиться что все внешние зависимости работают до написания кода.

- [ ] Проверить Циан API вручную — получить данные объекта по URL
- [ ] Проверить Циан API вручную — получить аналоги по параметрам
- [ ] Проверить 2GIS или Яндекс Карты API — получить инфраструктуру по координатам
- [ ] Проверить Claude API — простой вызов работает
- [ ] Зафиксировать rate limits всех API
- [ ] Зафиксировать структуру response каждого API

**Готово когда:** все API отвечают, структуры данных понятны.

---

### Phase 1 — Retrieval Layer
**Цель:** `cian_client.py` и `geo_client.py` возвращают чистые данные.

- [ ] `cian_client.py` — парсинг объекта по URL (адрес, площадь, цена, этаж, тип)
- [ ] `cian_client.py` — поиск аналогов (радиус, тип, комнатность, площадь ±20%)
- [ ] `cian_client.py` — фильтрация: исключить дубли и устаревшие объявления
- [ ] `geo_client.py` — получить инфраструктуру по координатам объекта
- [ ] Логирование в оба модуля
- [ ] Retries при ошибках

**Готово когда:** оба модуля возвращают структурированный JSON.

---

### Phase 2 — Formula Layer
**Цель:** `market_scorer.py` считает позицию объекта на рынке.

- [ ] Цена/м² объекта и каждого аналога
- [ ] Медиана и среднее по выборке аналогов
- [ ] Отклонение объекта от медианы в %
- [ ] Перцентиль объекта в выборке
- [ ] Медианный срок экспозиции аналогов
- [ ] Автоматическое определение сегмента по цене/м²
- [ ] Базовый расчёт рекомендованной цены (от медианы с коэффициентами)

**Готово когда:** все числа считаются корректно, покрыты тестами.

---

### Phase 3 — AI Layer
**Цель:** `narrative_engine.py` генерирует нарратив без галлюцинаций.

- [ ] Промпт для интерпретации локации (слой 4) — строго на данных geo_client
- [ ] Промпт для экспертной оценки (слой 5) — строго на данных слоёв 1-4
- [ ] Адаптация нарратива под контекст A / B / C
- [ ] Промпт для финального текста отчёта (слой 6)
- [ ] Проверка: AI не генерирует факты без источника
- [ ] Humanizer на выходе narrative_engine

**Готово когда:** нарратив читается как профессиональный отчёт, нет AI-slop.

---

### Phase 4 — PDF Output
**Цель:** `report_renderer.py` собирает финальный PDF.

- [ ] Шаблон PDF: резюме / рынок / позиция / визуализация / район / адаптивная секция
- [ ] Таблица аналогов
- [ ] Позиция объекта на шкале цены (simple chart)
- [ ] Market positioning chart
- [ ] Числа в PDF берутся из `market_scorer.py`, не из нарратива
- [ ] Файл сохраняется в `reports/`

**Готово когда:** PDF выглядит профессионально, числа совпадают с расчётами.

---

### Phase 5 — QA Loop
**Цель:** `qa_card.py` закрывает feedback loop.

- [ ] После каждого PDF выводится feedback card в консоль
- [ ] Категории: Аналоги / Цена / Галлюцинация / Тон / Формулировки / Другое
- [ ] Routing отклонений задокументирован в CLAUDE.md

**Готово когда:** Мария может отметить проблему за 10 секунд.

---

### Phase 6 — First Calibration
**Цель:** первая калибровка после 10-20 реальных отчётов.

- [ ] Собрать feedback cards
- [ ] Проанализировать по слоям: где больше всего отклонений
- [ ] Скорректировать фильтры / формулы / промпты
- [ ] Зафиксировать изменения в `logs/calibration_log.md`

**Готово когда:** второй прогон лучше первого по всем категориям feedback.
