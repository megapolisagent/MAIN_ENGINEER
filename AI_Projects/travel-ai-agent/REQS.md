# Travel-agent — требования и примеры

Цель
- Создать агента, который по запросу пользователя формирует план поездки: список мест, ресторанов, кафе, достопримечательностей и прочих точек интереса с краткими описаниями и полезной мета-информацией.

MVP (минимальный набор)
- Поддержка запросов: город/регион, даты/длительность, туристический профиль (семья, бизнес, романтика, бюджет и т.д.).
- Возврат структурированного списка: название, категория, краткое описание, рейтинг (если есть), адрес/район, приблизительное время посещения, часы работы, примерная цена, ссылка/источник и расстояние/время от центра/от базовой точки.
- Формат вывода: человекочитаемый текст + JSON для интеграций.

Функциональные требования
- Парсинг пользовательских требований и приоритетов.
- Генерация сбалансированного маршрута/списка по категориям (еда, развлечения, культура, природа).
- Поддержка ограничений (время, бюджет, доступность для детей/инвалидов).
- Возможность ранжирования по релевантности, рейтингу и расстоянию.
- Моки для офлайн-режима, интеграция с реальными API потом.

Нефункциональные требования
- Ответ в пределах 2–5 секунд (MVP может быть медленнее при внешних API).
- Конфиденциальность: не сохранять личные данные без явного согласия.

Источники данных (приоритет для интеграции)
- OpenStreetMap / Nominatim
- Google Places API / Places Web
- Yelp / TripAdvisor / локальные каталоги
- Локальные базы (файлы, кэши, ручной контент)

Формат входа — примеры запросов
- "3 дня в Барселоне с семьей, хочется архитектуру, пляжи и хорошие рестораны. Бюджет средний."
- "Однодневная остановка в Копенгагене, интересуют кофейни и музеи современного искусства."

Формат выхода — пример JSON (схема)
{
  "trip": "Барселона, 3 дня",
  "items": [
    {"name":"Sagrada Familia","category":"Достопримечательность","desc":"Храм Гауди","rating":4.7,"address":"...","time":"1.5h","price":"€25","source":"Google"},
    ...
  ]
}

Критерии ранжирования
- Соответствие запросу, рейтинг, расстояние/время в пути, режим работы, уникальность опыта.

MVP-ограничения
- На первом этапе вернуть до 30 рекомендаций, сгруппированных по дням/районам.

Дальнейшие шаги
- Уточнить шаблоны пользовательских запросов и приоритеты ранжирования.
- Создать прототип-агента с моками и CLI.

Примеры запросов для тестирования
- "Париж, 5 дней, романтическая поездка, хорошие рестораны и прогулки"
- "Дубай, 2 дня, активный отдых, шоппинг и развлечения для взрослых"
