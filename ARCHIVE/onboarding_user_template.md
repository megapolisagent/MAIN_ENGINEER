# Onboarding — Шаблон для read-only пользователя
## Подключение к системам MAIN_ASSISTANT и MAIN_ENGINEER

> **Назначение:** даётся новому человеку, которому Мария дала read-доступ к репозиториям на GitHub (`megapolisagent/MAIN_ASSISTANT`, `megapolisagent/MAIN_ENGINEER`).
>
> **Что адаптировать перед отправкой:**
> - Шаг 5 «Первая фраза агенту» — подставить имя пользователя и описание задачи.
> - Раздел «Prerequisites» — оставить или убрать в зависимости от того, что уже установлено у пользователя.

---

## ЧТО ТЫ ПОЛУЧАЕШЬ

Доступ к двум AI-системам Марии:
- **MAIN_ASSISTANT** — стратегический собеседник (думать, обсуждать, спорить)
- **MAIN_ENGINEER** — инженерный собеседник (архитектура, технические решения)

Уровень доступа на GitHub — **Read**:
- ✓ читать, скачивать, клонировать
- ✗ отправлять изменения обратно в систему Марии (push заблокирован)

---

## БЕЗОПАСНОСТЬ — ЧТО ЗНАЧИТ READ-ДОСТУП

**Системы Марии на GitHub в безопасности.** Read-доступ блокирует `git push` — твои изменения никогда не попадут в её оригинал.

**Но твоя локальная копия после `git clone` — это обычные файлы у тебя на диске.** Операционная система их не блокирует. То есть ты технически можешь:
- случайно поправить файл в редакторе;
- попросить Claude Code что-то изменить внутри маминой папки.

Эти правки:
- останутся **только на твоём диске**;
- не попадут в систему Марии;
- могут создать конфликт при следующем `git pull`.

**Правило простое:** не редактируй ничего внутри `MAIN_ASSISTANT/` и `MAIN_ENGINEER/`. Свои файлы — в свою отдельную папку (см. Шаг 3).

## ШАГ 1. Принять приглашение

Тебе придёт email от GitHub:
> "Maria invited you to repository megapolisagent/MAIN_ASSISTANT"

Открой → нажми **Accept invitation**. Сделай то же для второго репозитория (`MAIN_ENGINEER`).

Если письма нет — зайди на github.com → колокольчик уведомлений в правом верхнем углу.

---

## ШАГ 2. Скачать системы

Открой терминал, перейди в папку, где хочешь хранить мамины системы (например, в Документы), и выполни:

```
git clone https://github.com/megapolisagent/MAIN_ASSISTANT.git
git clone https://github.com/megapolisagent/MAIN_ENGINEER.git
```

Появятся две папки на твоём диске.

---

## ШАГ 3. Создать свою папку для агентов

Рядом с мамиными папками — твоя собственная папка под эксперименты:

```
mkdir Projects
```

Внутри `Projects/` ты создаёшь СВОИХ агентов (например `TRAVEL_AGENT`, `STUDY_AGENT`, `CAREER_AGENT` — что нужно). Это твоя территория, делай что хочешь.

**Внутри `MAIN_ASSISTANT/` или `MAIN_ENGINEER/` ничего не создавай.**

---

## ШАГ 4. Запустить ассистента Марии

```
cd MAIN_ASSISTANT
claude
```

Откроется Claude Code прямо в маминой системе. Он автоматически прочитает её правила и подключится как её стратегический помощник.

---

## ШАГ 5. Первая фраза агенту (обязательно)

Шаблон:

> «Я **[имя пользователя, отношение к Марии]**. Сегодня делаю **[короткое описание задачи или эксперимента]**. Использую тебя как **[стратегического / инженерного]** собеседника. Я не Мария — учти это при ответах.»

**Зачем это важно:** система настроена работать с Марией как с владелицей. Её ассистент по умолчанию считает, что перед ним она. Без явного представления агент может путаться или отвечать в её стиле, что тебе не нужно.

---

## ШАГ 6. Если случайно поправил файл внутри маминой папки

Это не страшно. Сделай внутри маминой папки:

```
git restore .
```

Все файлы вернутся к маминой версии на GitHub. Это очищает **твою локальную копию**, мамин оригинал на GitHub в этой операции вообще не участвует.

Твои собственные файлы в `Projects/` это не трогает.

---

## ШАГ 7. Обновить версию, когда Мария что-то изменит в системе

```
cd MAIN_ASSISTANT
git pull
```

Подтянет последнюю версию с GitHub.

Если `git pull` выдал ошибку про *"local changes"* — значит ты случайно что-то изменила внутри маминой папки. Сделай `git restore .` и повтори `git pull`.

---

## ПРАВИЛА — коротко

- Маминых папок (`MAIN_ASSISTANT/`, `MAIN_ENGINEER/`) **не редактировать**.
- Свои агенты — **только в `Projects/`**.
- Если случайно поправила маминый файл → `git restore .`.
- Обновления от Марии — через `git pull`, не копированием вручную.
- Что-то непонятно — спросить Марию.

---

## ВОЗМОЖНОСТИ — что ты можешь делать

- Запускать `claude` в маминой папке и работать с её агентами как с консультантами.
- Создавать своих агентов в `Projects/` и развивать их как угодно.
- Сохранять результаты экспериментов у себя.
- Подтягивать обновления маминой системы через `git pull`.
- Использовать своих собственных MCP, скиллы, плагины — это твой Claude Pro, твой контур.
